## **Summary of Data Flow:**

1. **User Interface** → Upload/list/search/chat with documents
2. **Document Ingestion** → Parallel processing:
   - File upload via API (multiple files supported)
   - Rich metadata extraction (filename, date, author, pages, sections, tags, hierarchy, type)
   - Document parsing (PyMuPDF, LangChain loaders, Unstructured, optional LlamaParse)
3. **Structure-Aware Chunking** → Recursive splitting respecting headings/sections, configurable size/overlap, streaming processing, explicit hierarchy metadata, preserving lists and tables
4. **Embedding Generation** → Using nomic-embed-text or bge-m3, batch processing, CPU optimized
5. **Vector Database** → Chroma or LanceDB (lightweight) with metadata filtering (planning to migrate to pgvector later)
6. **Query Processing** → User question + metadata filters (ranking to be added later)
7. **Retrieval Stage** → Hybrid approach combining vector similarity + metadata prefiltering + hierarchy retrieval
8. **Context Builder** → Assembles top-k chunks with source metadata, page/section info, and hierarchy context for the LLM
